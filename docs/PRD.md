## 1. 产品概述

### 1.1 产品愿景

记账本是一款轻量级的个人财务管理工具，致力于帮助用户轻松记录日常收支，培养良好的理财习惯。通过简洁直观的交互设计，让繁琐的记账变得简单高效，成为用户管理个人财务的贴心助手。

产品以"简单即美"为设计理念，让记账成为用户愿意坚持的习惯，而非负担。

### 1.2 目标用户

| 用户群体 | 年龄范围 | 描述 | 核心诉求 |
|----------|----------|------|----------|
| 职场新人 | 25-30岁 | 初入职场，收入有限 | 培养记账习惯，控制不必要支出 |
| 家庭财务管理者 | 30-50岁 | 负责家庭日常开支管理 | 统筹家庭收支，清晰了解钱花在哪里 |
| 学生群体 | 18-24岁 | 有限的生活费 | 建立理财意识，学会预算管理 |
| 自由职业者 | 25-45岁 | 收入不固定 | 随时随地记录收支，清晰财务状况 |

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| **简单易用** | 3秒内完成一笔记账操作 |
| **数据清晰** | 收支分类明确，结余一目了然 |
| **隐私安全** | 数据本地存储，无需注册账号 |
| **随时可查** | 支持按日期、分类等多维度查询 |

|----|----------|--------|
| US-001 | 作为用户，我希望能够快速记录一笔支出，无需选择过多分类，以便在忙碌时也能坚持记账 | P0 |
| US-002 | 作为用户，我希望记录我的各项收入，以便于全面了解我的资金流入情况 | P0 |
| US-003 | 作为用户，我希望查看我的历史收支记录，了解我的消费习惯和财务状况 | P0 |
| US-004 | 作为用户，我希望了解我本月的收支概况，掌握本月的财务状况 | P0 |
| US-005 | 作为用户，我希望搜索历史记账记录，快速找到特定项目的所有收支 | P1 |

### 2.2 用户故事详述

#### US-001：快速记录支出

> **作为** 一名用户
> **我想要** 快速记录一笔支出
> **以便于** 追踪我的日常消费情况

**场景**：
- 早上买咖啡时，顺手打开App记录
- 超市结账后立即记录购物清单

**验收标准**：
- 可以输入支出金额
- 可以从预设列表中选择支出分类（可选）
- 可以设置支出日期（默认为当天）
- 可以添加备注说明（可选）
- 记录完成后显示确认信息
- 全流程不超过30秒

#### US-002：记录收入

> **作为** 一名用户
> **我想要** 记录我的各项收入
> **以便于** 全面了解我的资金流入情况

**验收标准**：
- 可以输入收入金额
- 可以从预设列表中选择收入分类
- 可以设置收入日期（默认为当天）
- 可以添加备注说明（可选）
- 记录完成后更新账户结余显示

#### US-003：查看收支记录

> **作为** 一名用户
> **我想要** 查看我的历史收支记录
> **以便于** 了解我的消费习惯和财务状况

**验收标准**：
- 可以按日期范围筛选记录
- 可以查看收支明细列表
- 每条记录显示日期、分类、金额、备注
- 支持下拉刷新获取最新数据
- 列表按日期倒序排列

#### US-004：查看月度收支概况

> **作为** 一名用户
> **我想要** 了解我本月的收支概况
> **以便于** 掌握本月的财务状况

**验收标准**：
- 显示当月总收入、总支出、结余
- 可以切换查看不同月份的概况
- 收支数据以可视化方式呈现

#### US-005：搜索记账记录

> **作为** 一名用户
> **我想要** 搜索历史记账记录
> **以便于** 快速找到特定项目的所有收支

**验收标准**：
- 支持按关键词搜索备注内容
- 支持按日期范围筛选
- 支持按金额范围筛选
- 搜索结果准确展示匹配记录

## 3. 功能需求

### 3.1 P0 功能（必须实现）

| 功能编号 | 功能名称 | 功能描述 | 验收标准 |
|----------|----------|----------|----------|
| FR-001 | 新增支出记录 | 用户创建一条新的支出记录 | 1. 可输入金额<br>2. 可选择分类<br>3. 可设置日期（默认当天）<br>4. 可添加备注<br>5. 保存后显示成功提示 |
| FR-002 | 新增收入记录 | 用户创建一条新的收入记录 | 1. 可输入金额<br>2. 可选择分类<br>3. 可设置日期（默认当天）<br>4. 可添加备注<br>5. 保存后更新结余显示 |
| FR-003 | 收支记录列表 | 显示所有收支记录 | 1. 按日期倒序排列<br>2. 显示日期、分类、金额、类型<br>3. 支持下拉刷新 |
| FR-004 | 月度收支概况 | 查看指定月份的收支汇总 | 1. 显示当月总收入、总支出、结余<br>2. 支持切换月份<br>3. 显示收支分类占比 |
| FR-005 | 结余计算 | 系统自动计算当前结余 | 1. 结余 = 总收入 - 总支出<br>2. 首页始终可见<br>3. 变更后自动重新计算 |

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| **应用启动时间** | 冷启动不超过3秒 |
| **记账操作响应** | 保存操作完成后1秒内显示结果 |
| **列表加载** | 1000条记录内列表滑动流畅 |
| **统计计算** | 月度统计计算不超过1秒 |
| **数据查询** | 按日期筛选操作不超过500ms |
| **内存占用** | 运行时内存不超过150MB |

### 4.2 兼容性要求

| 项目 | 要求 |
|------|------|
| **iOS版本** | 支持iOS 14.0及以上 |
| **Android版本** | 支持Android 8.0 (API 26)及以上 |
| **屏幕适配** | 支持iPhone和Android主流屏幕尺寸 |
| **横竖屏** | 仅支持竖屏模式 |

### 4.3 数据存储要求

| 项目 | 要求 |
|------|------|
| **存储位置** | 本地设备存储（SQLite数据库） |
| **数据格式** | 结构化数据存储 |
| **备份方式** | 支持手动导出JSON格式备份 |
| **数据迁移** | 支持导入JSON格式恢复数据 |
| **无网络依赖** | 全部功能在离线状态下可用 |
| **存储容量** | 单条记录大小不超过1KB，支持至少10万条记录 |

### 4.4 安全要求

| 项目 | 要求 |
|------|------|
| **隐私保护** | 无需注册账号，不收集用户个人信息 |
| **数据隔离** | 数据仅存储在用户设备上 |
| **数据安全** | 用户数据不上传服务器 |

### 4.5 可用性要求

| 项目 | 要求 |
|------|------|
| **界面语言** | 简体中文 |
| **货币单位** | 人民币（元），精确到分 |
| **日期格式** | YYYY-MM-DD |
| **错误处理** | 操作失败时显示友好提示，不闪退 |

## 5. 术语表

| 术语 | 英文 | 定义 |
|------|------|------|
| **收入** | Income | 用户收到的资金，包括工资、兼职报酬、投资收益等各种来源的现金流入 |
| **支出** | Expense | 用户花费的资金，包括日常消费、购物、交通等各项现金流出 |
| **结余** | Balance | 用户在某一时间点的财务状况，计算公式为：结余 = 累计收入 - 累计支出 |
| **分类** | Category | 用于区分收支类型的标签，如"餐饮"、"交通"属于支出分类，"工资"、"奖金"属于收入分类 |
| **备注** | Note | 用户为单笔记录添加的说明文字，用于补充描述该笔收支的详细信息 |
| **账单** | Record | 记录的具体每一笔收支明细 |
| **月统计** | Monthly Summary | 对一个月内所有收支数据的汇总分析 |
| **数据导出** | Data Export | 将本地数据备份为独立文件的过程 |
| **数据导入** | Data Import | 从备份文件恢复数据的过程 |
| **P0功能** | Must Have | 产品必须实现的核心功能，是产品的基础 |
| **P1功能** | Should Have | 重要功能，计划在P0完成后实现 |
| **P2功能** | Nice to Have | 增强功能，优先级较低，可在后续版本迭代 |
